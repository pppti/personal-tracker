# 护肤品库存管理系统 - 公司内网部署说明

## 推荐结论

公司内网多人使用，请用 Docker 版。Docker 版是 Flask + MySQL，所有同事访问同一个数据库，支持产品新增/编辑/停用、入库、出库、盘点调整、统计分析和 CSV 导出。

- Web 管理界面：`http://公司电脑IP:5000`
- phpMyAdmin 数据库管理：`http://公司电脑IP:8080`
- MySQL：`公司电脑IP:3306`

如果只是先在一台电脑上试用，可以双击 `start.bat` 选 `1`，使用 SQLite 单文件版。SQLite 版不建议作为公司多人正式使用。

## 公司电脑需要安装什么

### Docker 正式版

在作为服务器的公司电脑上安装：

1. Docker Desktop
2. Windows 允许 Docker Desktop 启动
3. 把整个 `intranet` 文件夹复制到公司电脑，例如：

```bat
D:\vs-claude\intranet
```

这个文件夹已经包含 Docker 初始化数据库需要的 SQL 文件：

- `schema.sql`
- `seed.sql`
- `views.sql`
- `exports.sql`

### SQLite 试用版

只需要安装 Python 3.10 或以上版本，不需要安装数据库。

## Docker 正式启动步骤

在公司电脑打开 PowerShell 或 CMD：

```bat
cd /d D:\vs-claude\intranet
docker compose up -d
```

启动后，本机访问：

```text
http://localhost:5000
```

同事访问：

```text
http://公司电脑IP:5000
```

查看公司电脑 IP：

```bat
ipconfig
```

找到当前网卡的 `IPv4 地址`，例如 `192.168.1.25`，同事就访问：

```text
http://192.168.1.25:5000
```

## 防火墙设置

如果同事打不开，通常是 Windows 防火墙拦住了端口。

在作为服务器的公司电脑上，用管理员 PowerShell 执行：

```powershell
New-NetFirewallRule -DisplayName "Skincare Web 5000" -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow
New-NetFirewallRule -DisplayName "Skincare phpMyAdmin 8080" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow
```

一般只需要给同事开放 `5000`。`8080` 是数据库管理页面，建议只给管理员使用。

不建议把 `3306` MySQL 端口开放给所有同事，除非有外部系统确实需要直连数据库。

## 停止和重启

停止：

```bat
cd /d D:\vs-claude\intranet
docker compose down
```

重启：

```bat
cd /d D:\vs-claude\intranet
docker compose restart
```

查看运行状态：

```bat
docker compose ps
```

查看日志：

```bat
docker compose logs -f
```

## 数据保存在哪里

Docker 版数据保存在 Docker volume：`intranet_mysql_data`。

执行 `docker compose down` 不会删除数据。

不要执行下面这种命令，除非你明确要清空数据库：

```bat
docker compose down -v
```

## 备份数据库

在公司电脑执行：

```bat
cd /d D:\vs-claude\intranet
docker exec skincare-mysql mysqldump -uskincare -pskincare2025! skincare_inventory > backup.sql
```

恢复备份：

```bat
cd /d D:\vs-claude\intranet
type backup.sql | docker exec -i skincare-mysql mysql -uskincare -pskincare2025! skincare_inventory
```

## SQLite 单文件启动

双击：

```text
start.bat
```

选择：

```text
1
```

或者手动执行：

```bat
cd /d D:\vs-claude\intranet
python app_sqlite.py
```

SQLite 版数据库文件会生成在：

```text
D:\vs-claude\intranet\skincare.db
```

SQLite 适合个人或临时试用。多人长期同时使用，建议换 Docker + MySQL。

## 账号和安全建议

当前 Docker 默认数据库密码是：

```text
skincare2025!
```

正式放公司内网前，建议修改 `docker-compose.yml` 里的这些密码：

- `MYSQL_ROOT_PASSWORD`
- `MYSQL_PASSWORD`
- `DB_PASSWORD`
- `PMA_PASSWORD`

改完后第一次启动会使用新密码初始化数据库。已经启动过并产生数据后，再改密码需要同步修改 MySQL 用户密码。

## 常见问题

### 本机能打开，同事打不开

检查三件事：

1. 同事访问的是 `http://公司电脑IP:5000`，不是 `localhost`
2. 公司电脑和同事电脑在同一个局域网
3. 公司电脑 Windows 防火墙已放行 TCP `5000`

### Docker 启动失败，提示找不到 SQL

确认 `intranet` 目录里有：

```text
schema.sql
seed.sql
views.sql
exports.sql
```

### 端口被占用

如果 `5000`、`8080` 或 `3306` 已被其它软件占用，可以修改 `docker-compose.yml` 的 `ports`。

例如把 Web 改为 `5050`：

```yaml
ports:
  - "5050:5000"
```

同事访问就变成：

```text
http://公司电脑IP:5050
```
